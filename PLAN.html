Data flow
  1. On load: useTimezoneCity returns a default city (e.g., America/Chicago → Chicago). In parallel, SearchOverlay opens (dismissable).
  2. useWeather(city, units) fetches current + 7-day forecast from Open-Meteo in a single request.
  3. Response cached in a lightweight state + sessionStorage pattern.
  4. Unit toggle re-formats client-side (Open-Meteo returns metric; conversion is trivial). No refetch.
  5. Search hits geocoding → picks first result → sets city → refetch.

Phase 1 — Data + Default City Real weather on screen

Get real forecast data for a real default city with proper loading/error states. No polish, no tiles beyond the hero.

Steps

  1. api/openMeteo.ts — getForecast({ lat, lng, timezone }) hits /v1/forecast for current + hourly + daily + wind + UV + precipitation.
  2. api/geocoding.ts — searchCities(query) and getCityByName(name).
  3. api/types.ts — raw API + normalized shape used by components.
  4. lib/wmoCodes.ts — map WMO codes 0–99 to { label, iconKey, isRainy, isSunny, ... }.
  5. lib/formatters.ts — formatTemp, formatWind, formatTime, etc.
  6. hooks/useTimezoneCity.ts — Intl.DateTimeFormat().resolvedOptions().timeZone → curated { zone → city } table.
  7. hooks/useWeather.ts — { city, units } → { data, loading, error, refetch }.
  8. Hero.tsx — big temp, condition, high/low. Uses gradient.ts for weather-tinted green.
  9. gradient.ts — condition → Tailwind gradient class (always in the green family).
  10. Wire into App.tsx — default city → useWeather → <Hero />. Loading and error states render correctly.
  11. Unit toggle — useUnitStore, flips value, persists to localStorage.

Verification

  • App loads with real forecast for timezone-guessed city.
  • Unit toggle flips °C ↔ °F everywhere.
  • Preference persists on reload.
  • Killing the network shows error state, not a crash.
  • Green gradient visibly shifts across sunny / rainy / cloudy.

Phase 3 — Search + Recents Discoverability

First-load overlay + recent-cities chips + LRU logic.

Steps

  1. SearchOverlay — Shadcn Dialog, ESC/backdrop/X dismissable. Opens on first load; reopens via header search icon.
  2. SearchBar — 250ms debounced, geocoding API, top 5 matches with country.
  3. RecentCities — chip row, click loads, × removes.
  4. useRecentCitiesStore — LRU, max 5, localStorage. Existing city bumps to front.
  5. Input validation — trim, cap 100 chars, reject junk. Shows EmptyState for "type to search".

Verification

  • First-load: overlay appears, dismissable, default city loaded in background.
  • Search picks city → hero + tiles re-render.
  • Recent chip appears, click loads, × removes.
  • Max 5, 6th push evicts oldest.
  • Preferences persist across reload.

Phase 5 — Testing Comprehensive

We picked the bias-toward-more scope, so this phase is real work — not a checkbox.

Unit tests (Vitest)

  1. Every component in components/ has a test file.
  2. Every hook has a test file with mocked API.
  3. Formatters: table-driven test.
  4. WMO code map: assert every code has a label + icon.
  5. Recent-cities LRU: add, evict, dedupe-bumps, remove.
  6. Input validation: empty, whitespace, length cap, injection-ish.

Integration tests (Vitest + jsdom)

  1. useWeather + mocked fetch: happy, network error, empty, malformed.
  2. SearchOverlay full interaction: open, type, pick, close, city updates.

E2E tests (Playwright)

  1. Happy path: load → hero → search "Paris" → forecast updates → toggle °F → dark mode → reload → state persists.
  2. Error states: route.abort → error copy. City-not-found copy.
  3. A11y: @axe-core/playwright on landing + after-search.

Verification

  • npm run test — all pass, no .skip, no console errors.
  • npm run test:e2e — deterministic (3× locally).
  • Coverage report generated; gaps noted in CLAUDE.md.

Risks & Assumptions

  • Open-Meteo uptime. Free service, no SLA. If it flakes during ship, fallback to last-good cached response.
  • Timezone → city coverage. Curated table won't cover every timezone; fallback = Chicago. Acceptable for v1.
  • Meteocons packaging. If a community bundle doesn't play nice with Vite, fall back to raw SVGs in public/icons/.
  • Recharts bundle size. ~60KB gzipped. Fine for one chart; revisit if charts multiply.
  • Netlify free tier. 100GB bandwidth / 300 build min per month. Way more than needed.
